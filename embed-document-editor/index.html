Embed a Private Document Editor

Use an iframe and postMessage API to add local Office editing to your own web application — no backend, no upload, no Document Server required.

How Embedding Works

Point an <iframe> at https://bybrowser.com/?embed=1 and control the editor from your parent page using window.postMessage. Your application handles authentication and file storage; the iframe handles editing only. Because all processing happens in the browser, no document content ever passes through a server.

Quick Integration Example

<iframe id="editor" src="https://bybrowser.com/?embed=1"
  style="width:100%;height:720px;border:0"></iframe>

<script>
const editor = document.getElementById('editor').contentWindow;

// Open a document from URL
editor.postMessage(
  { type: 'document:open-url', payload: { url: 'https://example.com/doc.docx' } },
  'https://bybrowser.com'
);

// Listen for save events
window.addEventListener('message', (e) => {
  if (e.data?.type === 'document:saved') uploadFile(e.data.payload.file);
});
</script>

Supported Message Types

  • document:open-url — open a document from a URL
  • document:open — open from a File, Blob, or ArrayBuffer
  • document:save — trigger save and receive the file back
  • document:set-readonly — switch the editor to read-only mode
  • document:get-state — query current editor state

Frequently Asked Questions

Do I need to run a backend server?
No. The document editor is entirely client-side. Your backend only needs to serve or receive the file — editing and rendering happen in the iframe.
Can I restrict which origins can send messages?
Yes. Add ?embedOrigin=https://yourapp.com to the iframe URL to limit postMessage communication to your domain only.
What file formats can the embedded editor handle?
DOCX, XLSX, PPTX, and CSV are all supported in embedded mode.
