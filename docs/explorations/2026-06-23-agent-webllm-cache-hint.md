# 2026-06-23 Agent：本地模型缓存说明 + 缓存感知提示

回答"本地模型加载一次,刷新页面就不用再下了吧"——是的,并把这点在 UI 里讲清楚。

## WebLLM 缓存行为(确认)

- WebLLM 默认把模型权重存进浏览器 **Cache API**(持久化)。**首次下载后,刷新页面不会重新下载**,权重从缓存读。
- 但内存里的引擎(加载进 GPU 的那份)刷新后会丢,需要从缓存**重新加载一次**——这步很快(几秒、纯本地、无网络),与首次下载(~1.8GB)不是一回事。
- 所以刷新后再点「加载模型」或直接发送,是**从缓存秒开**,不是重新下载。

## 改动

- `llm/webllm.ts` 新增 `isModelCached(modelId)`:动态 import `@mlc-ai/web-llm` 的 `hasModelInCache` 检测某模型是否已缓存(SDK 加载失败/异常则返回 false)。与 engine 创建一样属于"需真实 SDK"的部分,不单测。
- `ui/panel.ts`:本地模式下按所选模型异步检测缓存,提示分两种:
  - 已缓存 →「该模型已缓存,点击「加载模型」秒开(刷新页面也不会重新下载)。」
  - 未缓存 →「首次使用需下载(~X GB),之后浏览器缓存,刷新不再下载。」
  - 切换模型也会重新检测并重建 controller。
