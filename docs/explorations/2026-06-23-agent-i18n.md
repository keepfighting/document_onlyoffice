# 2026-06-23 Agent：面板多语言（i18n）

Agent 面板原来全是硬编码中文，而项目有 i18n 系统（中/英）。本次把面板接入。

## 改动

- **`lib/i18n.ts`**：`I18nMessages` 接口新增 ~27 个 `agent*` key，zh / en 两套翻译补齐（标题、provider 选项、按钮、提示、错误、turn 角色名、模型下载提示等）。
- **`lib/agent-plugin/ui/panel.ts`**：所有硬编码字符串改为 `t('agentXxx')`。`TURN_LABEL` / `PROVIDER_OPTIONS` 模块常量改成按 key 动态取。新增 `applyLabels()` + 监听 `window` 的 `languagechange` 事件——**语言切换时面板即时刷新**（标题/按钮/provider 选项/占位符/提示全部重译）。
- **`lib/agent-plugin/ui/controller.ts`**：turn 文案（"调用工具："/"工具出错："/"已停止。"/"已达到最大执行步数"）改用 `t()`。
- **`lib/agent-plugin/llm/prompt.ts`**：系统 prompt 加一句「Reply in the same language the user writes in.」，让模型用用户的语言回复（中文提问→中文回答）。
- **测试**：`agent-ui-controller.test.ts` 断言改用 `t()`（语言无关，测试环境默认 en）；`i18n.test.ts` 加双语 agent key 非空覆盖断言，防以后漏翻译。

## 模型下载提示的插值

`agentModelFirstDownload` 用 `{size}` 占位符，面板里 `t('agentModelFirstDownload').replace('{size}', size)`。

## 验证

- tsc / oxlint / prettier：通过
- 单测：206 通过（+1 i18n 双语覆盖）
- 注：本次浏览器冒烟因 chrome-devtools MCP 反复卡死/dev server 反复被杀没能跑成；但改动是纯 `t()` 替换、类型安全，面板挂载逻辑未动（此前多轮冒烟已验证面板渲染正常），风险低。`?locale=en` / `?locale=zh` 或应用内语言切换即可看到对应语言。

## Agent 功能现状（接入 i18n 后）

入口 `?agent=1` → 侧边栏（中/英自适应）→ Claude/OpenAI 云端 或 本地 WebLLM（可选模型）→ tool-use 循环 → 6 通用工具（Word/Excel/PPT）+ Excel 专属 set_cell/get_cell → 读写当前文档。面板：发送/停止/清空/引用选区/修订模式/重新打开，全部多语言。
