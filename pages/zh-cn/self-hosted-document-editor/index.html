自部署文档编辑器

将浏览器端 Office 文档编辑器作为静态文件或 Docker 容器部署到自己的服务器，无需外部云依赖，无需订阅费用。

部署方式

  • 静态托管 — 执行 pnpm build 一次构建，将 dist/ 目录上传到任意静态托管平台（Nginx、Cloudflare Pages、Vercel、Netlify、S3）
  • Docker — 拉取 ghcr.io/ranuts/document:latest，一条命令即可运行
  • 内网隔离环境 — 从内部主机提供服务，初始部署后无需外网访问

Docker 快速启动

# 基础启动
docker run -d --name document -p 8080:80 ghcr.io/ranuts/document:latest

# 启用 HTTPS
docker run -d --name document -p 443:443 \
  -v /path/to/certs:/ssl \
  -e SERVER_HTTP2_TLS=true \
  -e SERVER_HTTP2_TLS_CERT=/ssl/cert.pem \
  -e SERVER_HTTP2_TLS_KEY=/ssl/key.pem \
  ghcr.io/ranuts/document:latest

为什么选择自部署

  • 所有文档处理保留在自己的网络边界内
  • 不依赖第三方服务和其 SLA 保障
  • 完全掌控版本更新
  • 适用于禁止使用外部 SaaS 工具的合规环境

常见问题

服务器有什么配置要求？
任何能够提供静态文件的主机均可——整个应用由 HTML、CSS、JavaScript 和 WebAssembly 组成，运行时无需 Node.js 或数据库。
如何升级到新版本？
拉取最新 Docker 镜像，或从源码重新执行 pnpm build 并重新部署 dist/ 目录。
可以在前面加认证吗？
可以。使用 Nginx basic auth、Cloudflare Access 或任何支持认证的反向代理来保护编辑器。Docker 镜像支持 SERVER_BASIC_AUTH 配置 BCrypt 密码。
自部署需要许可证吗？
ByBrowser 采用 AGPL-3.0 协议，私人自部署使用无需付费。如果你分发修改后的版本，需要以相同协议开源你的修改。
