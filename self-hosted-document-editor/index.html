Self-hosted Document Editor

Deploy a browser-based Office document editor as static files or a Docker container on your own infrastructure — no external cloud dependency, no subscription.

Deployment Options

  • Static hosting — build once with pnpm build, upload the dist/ folder to any static host (Nginx, Cloudflare Pages, Vercel, Netlify, S3)
  • Docker — pull ghcr.io/ranuts/document:latest and run in one command
  • Air-gapped networks — serve from an internal host with no external internet access required after initial setup

Docker Quick Start

# Basic
docker run -d --name document -p 8080:80 ghcr.io/ranuts/document:latest

# With HTTPS
docker run -d --name document -p 443:443 \
  -v /path/to/certs:/ssl \
  -e SERVER_HTTP2_TLS=true \
  -e SERVER_HTTP2_TLS_CERT=/ssl/cert.pem \
  -e SERVER_HTTP2_TLS_KEY=/ssl/key.pem \
  ghcr.io/ranuts/document:latest

Why Self-host?

  • Keeps all document processing inside your own network perimeter
  • No dependency on third-party services or uptime SLAs
  • Full control over updates and version pinning
  • Suitable for compliance environments where external SaaS tools are restricted

Frequently Asked Questions

What are the server requirements?
Any host that can serve static files works — the entire application is HTML, CSS, JavaScript, and WebAssembly. No Node.js or database is required at runtime.
How do I update to a newer version?
Pull the latest Docker image or rebuild from source with pnpm build and redeploy the dist/ folder.
Can I add authentication in front of it?
Yes. Use Nginx basic auth, Cloudflare Access, or any reverse proxy with authentication to protect the editor. The Docker image supports SERVER_BASIC_AUTH for BCrypt-hashed passwords.
Is a license required to self-host?
ByBrowser is AGPL-3.0 licensed. Self-hosting for private use is permitted. If you distribute a modified version, you must open-source your changes under the same license.
