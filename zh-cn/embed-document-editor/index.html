嵌入隐私文档编辑器

通过 iframe 和 postMessage API 将本地文档编辑器集成到你自己的 Web 应用——无需后端服务器，无需上传，无需 Document Server。

嵌入原理

将 <iframe> 指向 https://bybrowser.com/?embed=1，通过 window.postMessage 从父页面控制编辑器。你的应用负责处理认证和文件存储，iframe 只负责编辑。由于所有处理均在浏览器内完成，文档内容不会经过任何服务器。

快速集成示例

<iframe id="editor" src="https://bybrowser.com/?embed=1"
  style="width:100%;height:720px;border:0"></iframe>

<script>
const editor = document.getElementById('editor').contentWindow;

// 从 URL 打开文档
editor.postMessage(
  { type: 'document:open-url', payload: { url: 'https://example.com/doc.docx' } },
  'https://bybrowser.com'
);

// 监听保存事件
window.addEventListener('message', (e) => {
  if (e.data?.type === 'document:saved') uploadFile(e.data.payload.file);
});
</script>

支持的消息类型

  • document:open-url — 从 URL 打开文档
  • document:open — 从 File、Blob 或 ArrayBuffer 打开
  • document:save — 触发保存并接收文件回调
  • document:set-readonly — 切换编辑器为只读模式
  • document:get-state — 查询当前编辑器状态

常见问题

需要运行后端服务器吗？
不需要。文档编辑器完全在客户端运行，你的后端只需负责文件的提供和接收，编辑和渲染在 iframe 内完成。
可以限制哪些来源可以发送消息吗？
可以。在 iframe URL 中添加 ?embedOrigin=https://yourapp.com，将 postMessage 通信限制为你的域名。
嵌入编辑器支持哪些文件格式？
嵌入模式支持 DOCX、XLSX、PPTX 和 CSV。
