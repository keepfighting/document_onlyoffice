OnlyOffice WASM Document Editor

A pure static web app that runs the OnlyOffice document editing engine in the browser via WebAssembly — no Document Server, no backend, no upload required.

What Is OnlyOffice WASM?

OnlyOffice is a full-featured open-source office suite. Normally it requires a dedicated Document Server to handle file processing. ByBrowser compiles the OnlyOffice x2t conversion engine to WebAssembly (WASM) and loads the web-apps frontend directly in the browser, eliminating the need for any server infrastructure. The result is a fully functional Office editor that runs in a static HTML page.

How the Architecture Works

  • OnlyOffice web-apps provide the editor UI (document, spreadsheet, presentation editors)
  • x2t WASM handles format conversion between OOXML formats (DOCX, XLSX, PPTX) and OnlyOffice's internal format
  • No Document Server — the socket.io backend is replaced with a local fallback so the editor loads in desktop-editor mode
  • Service Worker caches assets for repeat visits and PWA installation

Use Cases

  • Developers who want to embed a document editor without operating a backend server
  • Organizations that need an air-gapped document editing solution
  • Anyone researching how OnlyOffice can be run as a pure client-side application

Frequently Asked Questions

What is the difference between this and OnlyOffice Document Server?
OnlyOffice Document Server runs format conversion on the server. ByBrowser performs all conversion in the browser using WebAssembly, with no server required.
Is this an official OnlyOffice product?
No. ByBrowser is an independent open-source project that builds on OnlyOffice's open-source components (web-apps, sdkjs, x2t-wasm).
Can I self-host this?
Yes. The built output is a folder of static files that you can serve from any static host — Nginx, Cloudflare Pages, Vercel, or a Docker container.
